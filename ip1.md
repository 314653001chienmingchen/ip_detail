## 1. IP 是什麼？

**IP（Internet Protocol Address，網際網路協定位址）** 是設備在網路世界中的「實體地址」。不論是實體伺服器、虛擬機還是個人電腦，只要需要聯網通訊，就必須配置一個 IP。

> 💡 **對應比喻**：
> * 實體房屋 ➡️ 擁有**郵寄地址**
> * Linux 伺服器 / VMware 虛擬機 ➡️ 擁有 **IP 位址**（例如：`192.168.1.50`）
>
> 當設備需要傳送資料給另一台設備時，網路設備會透過 IP 位址精準找到目的端。

---

## 2. IPv4 與 IPv6 的差異與現況

目前主流網路環境主要分為兩種 IP 版本：

### 2.1 IPv4
IPv4 是目前企業環境**最常見、應用最廣泛**的 IP 格式。
* **格式**：由四組 0 ~ 255 的數字組成，中間以點號分隔。
* **結構表示**：`xxx.xxx.xxx.xxx`
* **範例**：`172.23.181.185`

```text
 172   .   23   .  181   .  185
  │         │        │        │
第 1 組   第 2 組   第 3 組  第 4 組
```

| IP 類型 | 範例 | 說明 |
| --- | --- | --- |
| **合法 IP** | `192.168.1.100`、`10.0.0.1`、`172.16.5.20` | 每組數字均嚴格在 `0 ~ 255` 範圍之內。 |
| **非法 IP** | `300.1.1.1`、`192.168.1.999` | 數值超出單組 8 bits (255) 的定義限制。 |

### 2.2 IPv6

為了解決 IPv4 位址數量有限、即將枯竭的問題而發展出來的全新版本。

* **格式**：使用十六進位制，由冒號分隔。
* **範例**：`2001:0db8:85a3::8a2e:0370:7334`
* **現況**：雖然 Internet（外網）正逐漸普及 IPv6，但**企業內部區域網路（LAN）目前仍大量依賴 IPv4**。

## 3. IP 位址結構（Network Part vs Host Part）

一個 IP 位址在邏輯上可以拆分為兩個核心部分：

1. **Network Part（網路部分）**：代表設備屬於哪一個聯網區域（網段）。
2. **Host Part（主機部分）**：代表該區域中的特定設備編號。

```ini
範例 IP：192.168.10.50 /24

   192.168.10     │        50
──────────────────┼──────────────────
  Network Part    │    Host Part
 (網路識別/網段)   │  (主機識別/設備)
```

* **意義**：這代表該設備身處在 `192.168.10` 這個網路區域中，而它的設備代號是 `50`。

---

## 4. Subnet Mask（子網路遮罩）的作用

Subnet Mask（子網路遮罩）是一把用來切割 IP 的「尺」，它的核心用途是：

> 告訴作業系統與網路設備，IP 裡面有**哪些部分屬於網路（網段）**，**哪些部分屬於主機（設備）**。

* **傳統寫法**：IP `192.168.1.50` 搭配 Mask `255.255.255.0`
* **現代斜線表示法 (CIDR)**：`192.168.1.50/24`
* **運作邏輯**：`/24` 代表前 24 個 Bits（即二進位中的 24 個 1，換算成十進位就是前三組數字 `192.168.1`）全部被鎖定為網路範圍，最後一組數字 `50` 則是 Host 空間。

---

## 5. CIDR 表示法與網路範圍（/8、/16、/24）

**CIDR (Classless Inter-Domain Routing)** 利用斜線法來快速定義子網路遮罩長度與容納主機數：

### 5.1 /24

* **範例**：`192.168.1.0/24`
* **範圍**：`192.168.1.1` ~ `192.168.1.254`
* **可用主機**：扣除網路位址（.0）與廣播位址（.255）後，可容納 **254** 台設備。常用於中小型辦公室、單一 VLAN 網段。

### 5.2 /16

* **範例**：`172.16.0.0/16`
* **範圍**：`172.16.0.1` ~ `172.16.255.254`
* **可用主機**：約可容納 **65,534** 台設備。常用於中大型企業的跨部門大網段。

### 5.3 /8

* **範例**：`10.0.0.0/8`
* **可用主機**：超大型架構，可容納上千萬台設備。大型企業、跨國集團或公有雲（AWS/Azure）底層骨幹常使用此網段。

## 6. Private IP（私有 IP）三大保留範圍

為了節約公網 IP 資源，企業內部網路（內網 / LAN）通常使用 **Private IP**。這些 IP 在 Internet 上是無法被直接路由的，主要分為三大標準範圍：

### 6.1 10.x.x.x

* **範圍**：`10.0.0.0/8`
* **範例**：`10.1.1.10`
* **特性**：容量極大，大型企業資料中心（Data Center）最愛用。

### 6.2 172.x.x.x

* **範圍**：`172.16.0.0/12`（包含 `172.16.x.x` 到 `172.31.x.x`）
* **範例**：`172.23.181.185`
* **特性**：中型企業愛用，**你的 VMware 虛擬機（VM）很可能就在這個範圍內**。

### 6.3 192.168.x.x

* **範圍**：`192.168.0.0/16`
* **範例**：`192.168.1.100`
* **特性**：最廣為人知，大量用於家用分享器、Wi-Fi 分享、小型辦公室。

## 7. Public IP（公網 IP）與企業網路架構

**Public IP（公網 IP / 外網 IP）** 是在全球 Internet 上公開、獨一無二且可以被直接找到的真實地址（例如 Google Public DNS 的 `8.8.8.8`）。

> 🌐 **企業標準網路傳遞架構模型**：

```text
 內部區域網路 (LAN)
 [ 內部 Server: 172.23.181.185 ]
               │
               ▼
 [ 企業防火牆 Firewall / NAT ] ──► (執行網路位址轉換)
               │
               ▼
 網際網路 (Internet)
 [ 公網 IP: 203.xxx.xxx.xxx ] ──► 外部使用者只會看到這個公網 IP
```

---

## 8. Gateway（閘道）的通訊流程

**Gateway（閘道 / 網關）** 是連通不同網路區域之間的「大門出口」。通常由路由器（Router）或防火牆（Firewall）的介面擔任。

* **情境 1：同網段直連**
VM (`172.23.181.185/24`) 想要連線到 `172.23.181.20`。因為在同一個 `/24` 網段內，**不需要經過 Gateway**，封包直接透過交換器直達。
* **情境 2：跨網段外聯**
VM 想要連線到 `8.8.8.8`。系統判斷 `8.8.8.8` 絕非同網段，因此**強制將封包送交給預設閘道 Gateway (`172.23.181.1`)** 轉發。

```text
[ Linux VM: 172.23.181.185 ]
             │ (目的地為 8.8.8.8，跨網段封包)
             ▼
[ Gateway: 172.23.181.1 ]
             │ (路由路徑尋找)
             ▼
[ 核心路由器 Core Router ] ──► [ Internet (8.8.8.8) ]
```

## 9. DNS 與 IP 的對應關係

由於人類的記憶力很難記住海量的 IP 數字組合，因此誕生了 **DNS（Domain Name System，網域名稱系統）**。

* **核心定位**：扮演網路世界的「電話簿」。
* **運作機制**：當你在瀏覽器輸入網址 `www.google.com` ➡️ 電腦向 DNS 伺服器發起查詢 ➡️ DNS 回報真實的伺服器 IP `142.250.xxx.xxx` ➡️ 電腦改用 IP 建立連線。

## 10. Linux 實戰：查看 IP 與 Routing Table

在 modern Linux 系統（如 CentOS 7/8/9, Ubuntu, Rocky Linux）中，請揚棄舊式的 `ifconfig`，全面改用強大的 `iproute2` 指令集：

### 🛠️ 查看網卡與 IP 位址

```bash
ip addr
```

* **輸出範例與重點拆解**：
```text
2: ens192: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...
    inet 172.23.181.185/24 brd 172.23.181.255 scope global ens192
```

> * **網卡名稱**：`ens192`（VMware 常見網卡命名）
> * **配置 IP**：`172.23.181.185`
> * **子網範圍**：`/24`（即 Mask `255.255.255.0`）

### 🛠️ 查看路由表（Routing Table / 確認 Gateway）

```bash
ip route
```

* **輸出範例與重點拆解**：
```text
default via 172.23.181.1 dev ens192 proto static metric 100
```

> * **預設閘道 (Default Gateway)**：`172.23.181.1`
> * **出接口網卡**：所有要出外網的封包，都會從 `ens192` 送出去給 `.1` 這台 Gateway。

## 11. VMware 環境中的虛擬網路架構訊號流

在 VMware vSphere 環境中，Linux 虛擬機（VM）的網路封包並非直接連到實體線，而是經過多層虛擬化元件才封裝送出：

```text
[ Linux 虛擬機 ]
       │ (透過 Virtual NIC / 如 vmxnet3 網卡驅動)
       ▼
[ vSwitch / vDVS (虛擬交換器) ]
       │ (身處在特定的 Port Group，並在此打上 VLAN Tag)
       ▼
[ Physical NIC / Uplink (實體網卡) ] ──► (如伺服器後方的刀鋒板網卡)
       │
       ▼
[ Physical Switch (實體核心交換器) ] ──► (必須設定為 Trunk Port 接收 Tag)
       │
       ▼
[ L3 Switch / Router (Gateway 閘道) ] ──► (進行跨網段路由與出外網)
```

## 12. 三劍客：VLAN、Routing 與 IP 的角色關係

在企業級網路架構中，這三者各司其職、緊密配合：

| 項目 | 核心用途 | 運作層級 | 實際工程任務 |
| --- | --- | --- | --- |
| **IP Address** | **設備的專屬地址** | Layer 3 (網路層) | 標示網路中單一端點的具體通訊位置。 |
| **VLAN** | **切割與隔離虛擬區域** | Layer 2 (資料連結層) | 阻絕廣播風暴。例如：將「開發部網段」與「財務部網段」做實體隔離，互不相通。 |
| **Routing** | **修築跨網路的橋樑** | Layer 3 (網路層) | 當 VLAN 10 想要與 VLAN 20 進行必要通訊時，由路由器負責安全轉發。 |

```text
 [ VLAN 10 架構 ]                         [ VLAN 20 架構 ]
 網段: 192.168.10.0/24                   網段: 192.168.20.0/24
        │                                        │
        └───────────────► [ 路由器 Router ] ◄────┘
                         (負責跨 VLAN 的 Routing)
```

## 13. 後續 Linux 網路進階學習地圖

### 13.1 建議學習順序

1. **Subnetting**：子網路切分、計算可用 IP、了解二進位轉換。
2. **ARP 與 MAC Address**：理解 Layer 2 實體定址與 IP 的轉換機制。
3. **Linux Network Interface**：配置靜態 IP（Static）與動態 IP（DHCP）設定檔。
4. **DNS 深度設定**：`/etc/resolv.conf` 與 `/etc/hosts` 的優先權。
5. **Linux 防火牆安全**：學習 `firewalld` 或 `iptables` 規則鏈控制。
6. **進階故障排查**：三層與四層網路連線分析。

### 13.2 系統維運必備核心網路指令

```bash
ip addr                # 查看與配置網路卡 IP 位址
ip route               # 查看與修改系統路由表（確認預設閘道）
ping -c 4 8.8.8.8      # 測試基礎 ICMP 連通性與延遲時間
ss -tunlp              # 查看本機目前開啟與監聽的 Port (全面取代舊的 netstat)
cat /etc/resolv.conf   # 檢查本機作業系統配置的 DNS 伺服器位址
nslookup google.com    # 測試 DNS 網域解析功能是否正常運作
traceroute 8.8.8.8     # 追蹤封包路由節點，找出網路斷點在內網、網關還是外網
```

## 💡 總結

* **IP** 決定你是誰，**Subnet** 決定你的內網朋友圈有多大。
* **Gateway** 是你走出朋友圈、面向世界（Internet）的必經大門。
* **VLAN** 在底層默默幫你把不同的房間隔開，而 **Routing** 則是為這些房間修築符合安全規範的聯絡通道。
