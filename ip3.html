Switch · 網路基礎觀念

交換機到底在做什麼事？

交換機（Switch）是區域網路裡最基礎的設備。它靠著記住「MAC 位址在哪個孔」，把封包精準送到該去的地方——而不是像集線器（Hub）那樣，收到什麼就往所有孔亂槍打鳥式地全部送出去。

Compare

交換機 vs 集線器 vs 路由器

三個常被搞混的設備，差別在於「依據什麼判斷該送去哪裡」。

Hub 集線器
Layer 1

沒有判斷能力，收到訊號就從所有孔全部送出去，所有裝置共用同一個碰撞網域。現在幾乎絕跡。

Switch 交換機
Layer 2

依 MAC 位址 判斷，查表後精準送到對的 port，是區域網路裡最常見的核心設備。

Router 路由器
Layer 3

依 IP 位址 判斷，負責跨網段轉送封包，決定封包該走哪條路由。

How it works

MAC 位址表是怎麼建立的

交換機開機時什麼都不知道，透過四個階段慢慢學會「誰在哪個孔」。

01 Flooding（洪泛）——MAC 表是空的，收到封包不知道目的地在哪個孔，就從來源孔以外的所有孔送出去，行為暫時很像 Hub。
02 Learning（學習）——同時把「這個封包的來源 MAC」跟「它是從哪個孔進來的」記錄進 MAC 位址表（也叫 CAM Table）。
03 Forwarding（轉送）——之後再收到要送去這個 MAC 的封包，直接查表從對應的孔送出去，不再往其他孔浪費頻寬。
04 Aging（老化）——一段時間（預設常見 300 秒）沒再看到某個 MAC 出現，就把它從表裡刪掉，避免記錄過期或表爆滿。

Domains

碰撞網域 vs 廣播網域

這是交換機能解決的問題，也是它跟 Hub 最根本的差異。

網域類型問題誰能切開
碰撞網域
Collision Domain		同一時間兩台裝置一起傳送會互相干擾（碰撞）Switch 的每個 port 都是獨立碰撞網域
廣播網域
Broadcast Domain		廣播封包會傳到網域內所有裝置，裝置一多流量就爆炸只有 VLAN 或 Router（Layer 3 裝置）能真正切開

關鍵觀念：交換機解決了碰撞網域的問題，但解決不了廣播網域的問題——同一台交換機（或用 Trunk 相連的多台交換機）上的所有裝置，預設都在同一個廣播網域裡。

VLAN

VLAN：把一台交換機切成好幾個邏輯網路

VLAN（Virtual LAN）讓你不用多買硬體，就能把同一台實體交換機邏輯上切成好幾個獨立的廣播網域。

Port 類型行為常見用途
Access Port只屬於單一 VLAN，進出的封包不帶 VLAN 標籤（untagged）接電腦、印表機等終端設備
Trunk Port允許多個 VLAN 的流量通過，封包會帶上 VLAN 標籤（tagged）交換機之間互連、接虛擬化主機
Loop Prevention

STP：避免多台交換機接成迴圈

如果為了備援把多台交換機用兩條線互連，會形成實體迴圈，廣播封包會被無限重複轉送、瞬間癱瘓整個網路（廣播風暴）。STP（Spanning Tree Protocol，生成樹協定）會自動選出一台 Root Bridge，並把多餘的路徑邏輯上「關閉」（Blocking），只留一條路徑通行，等主要線路斷了才會自動切換備援路徑。

Types

交換機的分類

分類說明
Unmanaged隨插即用，無法設定任何東西，常見於家用小型交換器
Managed可設定 VLAN、Port 鏡像、STP、SNMP 監控等，企業 / 機房標配
L2 Switch只依 MAC 位址轉送，不處理 IP 層的事情
L3 Switch除了 L2 功能，還內建路由能力，可以直接跨 VLAN 轉送（Inter-VLAN Routing），速度通常比外接路由器快
