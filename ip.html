Math

IP 位址計算

在動手輸入指令之前，先搞懂 IP 位址本身怎麼組成、怎麼分類、怎麼算子網路——這是所有 ip addr / ip route 操作的地基。

32 位元的結構

IPv4 位址是一組 32 位元二進位數字，習慣切成 4 組、每組 8 位元（1 個 octet），轉成 0–255 的十進位並用點分隔，這就是「點分十進位」（dotted decimal）。

1921681130
11000000101010000000000110000010

每一組 8 bit 都能表示 0～255（2⁸ = 256 種可能），四組合起來就是完整的 32 bit 位址。

IP 位址分類（Class）

依照最前面幾個位元，傳統上把 IP 分成 A～E 五個等級，決定了預設的網路 / 主機切法。

類別第一組範圍預設遮罩用途
A1 – 126255.0.0.0 (/8)大型網路
B128 – 191255.255.0.0 (/16)中型網路
C192 – 223255.255.255.0 (/24)小型網路
D224 – 239—多播 Multicast
E240 – 255—保留 / 實驗用

127 開頭是保留給 loopback（如 127.0.0.1），不屬於任何一般分類。現代網路多半改用不看 Class、直接寫前綴長度的 CIDR 表示法。

子網路遮罩與 CIDR

子網路遮罩（Subnet Mask）用來標出 32 bit 裡哪些是「網路位元」、哪些是「主機位元」。CIDR（如 /24）就是網路位元的數量，兩者是同一件事的兩種寫法。

計算公式

網路位址=IP AND 子網路遮罩
廣播位址=網路位址 OR 反遮罩（Wildcard）
可用主機數=2(32 − 前綴長度) − 2

扣掉的 2 個，是被保留起來的網路位址（全 0 主機位元）和廣播位址（全 1 主機位元），不能分給任何一台主機。

實際算一次：192.168.1.130 /26

關鍵那組 octet130 = 10000010　遮罩 192 = 11000000
AND 運算10000010 AND 11000000 = 10000000 = 128
網路位址192.168.1.128
廣播位址192.168.1.191
可用範圍192.168.1.129 ～ 192.168.1.190
可用主機數2⁶ − 2 = 62

Object

四大管理物件

語法固定為 ip [Object] [Command]，先認識四個物件，後面的指令都是它們的延伸。

address 簡寫 a

管理主機上的 IP 位址——新增、刪除、查看。

route 簡寫 r

管理路由表，決定封包該往哪個閘道 / 介面送出。

link 簡寫 l

管理網路卡（NIC）本身，例如啟用或停用介面。

neigh 簡寫 n

管理 ARP / Neighbor Table，也就是 IP 對應 MAC 的快取。

01 · Address

IP 位址管理

新增、刪除、查看主機介面上的 IP 位址。

02 · Route

路由管理

控制封包的下一步：預設閘道與靜態路由。

03 · Link

網卡管理

查看、啟用、停用網路介面。

04 · Neigh

ARP / Neighbor Table

查看區域網路中 IP 對應 MAC 的快取表。

Tip

推薦查看方式

彩色 + 精簡輸出

$ ip -c -br a
  • -c　彩色顯示，介面狀態一眼可辨
  • -br　簡潔模式（brief），每個介面一行
