v86 · 瀏覽器內的真實 Linux

在瀏覽器裡開一台真的 Linux

這頁用 v86（WebAssembly x86 模擬器）在瀏覽器裡直接跑一顆精簡 Linux 核心，你可以打真正的 ip 指令、看真正的核心回應，不是模擬假輸出。

部署前必讀：這個頁面本身不含 Linux 系統檔案（BIOS、核心映像檔約 10MB），需要照下面「部署步驟」自行下載並放進對應資料夾，否則畫面只會停在「載入中」或顯示錯誤。點開下方説明展開步驟。
部署步驟（第一次設定，約 10 分鐘）
  1. 在 terminal.html 同一層資料夾裡，建立一個 v86/ 子資料夾，資料夾結構最後要長這樣： 
    ip-guide/
├── index.html
├── terminal.html
└── v86/
    ├── libv86.js
    ├── v86.wasm
    ├── seabios.bin
    ├── vgabios.bin
    └── buildroot-bzimage68.bin
  2. 到 v86 官方 GitHub Releases 下載 libv86.js 與 v86.wasm：
    github.com/copy/v86/releases（下載最新版 release 裡附的 build 檔案）。
  3. BIOS 檔案（seabios.bin、vgabios.bin）在 v86 專案原始碼的 bios/ 資料夾裡，一併從同一個 release 或原始碼取得。
  4. 下載一顆精簡 Linux 核心映像（Buildroot，內建 BusyBox，含基本 ip 指令）： 
    curl -O https://i.copy.sh/buildroot-bzimage68.bin
    放進 v86/ 資料夾裡。
  5. 本機測試：不能直接雙擊開啟 HTML（瀏覽器會擋掉檔案讀取），要用簡單的本機伺服器跑： 
    cd ip-guide
python3 -m http.server 8000
    瀏覽器打開 http://localhost:8000/terminal.html。
  6. 確認能開機、能打指令之後，把整個 ip-guide/ 資料夾（含 v86/）一起上傳到 GitHub Pages / Netlify 之類的靜態空間，就完成部署了。
  7. 想要完整的 bridge / tc 指令？ Buildroot 內建的是 BusyBox 版 ip，功能較陽春。如果想要本網站教的橋接器、VLAN、tc 鏡像都能實測，改用 Alpine Linux 映像開機，進去後執行 apk add iproute2 bridge iproute2-tc 安裝完整工具即可，作法可參考 v86 官方文件的「Alpine Linux guest setup」。

開機後可以試試看

  • ip a — 查看目前的網路介面
  • ip addr add 192.168.1.100/24 dev eth0 — 加一個 IP
  • ip link set eth0 up / down — 啟用 / 停用網卡
  • ip route — 查看路由表
  • 這是真的核心，沒有連到外部網路，所有操作都只影響這台虛擬機自己的狀態，可以放心亂試、亂改。
